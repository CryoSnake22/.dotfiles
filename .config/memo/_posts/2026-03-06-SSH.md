# SSH

## Jumps

Apparently lots of networks function in the current way where the private network is only accessible from some jump host.

```
Internet
    │
    ▼
Bastion / Jump Host (public)
    │
    ▼
Private Network
    ├── database server
    ├── internal API
    └── admin dashboard
```

To connect you would have to first

```bash
ssh user@jump-host
# Then
ssh user@private-network
```

Which works but is quite clunky. The modern way to do it would be to do

```bash
ssh -J jump-server internal-server
# So it would look like:
ssh -J user@jump-server user@internal-server
```

You can also configure this in your `~/.ssh/config` with

```txt
Host jump
  HostName jump.example.com
  User alice

Host db
  HostName 10.0.0.5
  User alice
  ProxyJump jump
```
